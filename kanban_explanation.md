# Kanban Board Explanation – AccessLearn

## What is a Kanban Board?

A Kanban board is a visual project management tool that tracks tasks across different stages of development using columns such as "To Do", "In Progress", and "Done".

---

## How the Board Visualizes Workflow

The Kanban board represents workflow visually:

- "To Do" shows tasks not started
- "In Progress" shows active tasks
- "Done" shows completed tasks
- Additional columns such as "Testing" and "Blocked" improve tracking

---

## Work-In-Progress (WIP) Limits

The board limits work-in-progress by:

- Allowing only a few tasks in "In Progress"
- Preventing overload
- Helping focus on completing tasks before starting new ones

---

## Support for Agile Principles

The Kanban board supports Agile by:

- Enabling continuous delivery
- Allowing flexibility in task management
- Improving visibility of work
- Supporting iterative development

---

## Customization for AccessLearn

The board was customized by adding:

- "Testing" column for quality assurance
- "Blocked" column for issues preventing progress

This improves tracking and aligns with real-world development practices.
