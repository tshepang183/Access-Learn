# Template Analysis – AccessLearn

## Comparison of GitHub Project Templates

| Template | Columns / Workflow | Automation | Agile Suitability |
|----------|------------------|------------|-------------------|
| Kanban | To Do, In Progress, Done | Manual task movement | Simple Agile tracking |
| Bug Triage | New, Triaged, Fixed | Automation for bug labeling | Good for maintenance |
| Team Planning | Backlog, Ready, In Progress, Done | Partial automation | Good for structured planning |

---

## Selected Template: Kanban

### Justification

The Kanban template was selected because:

- It provides a simple and clear workflow structure (To Do, In Progress, Done)
- It supports Agile development through task visualization
- It allows easy customization by adding additional columns such as "Testing" and "Blocked"
- It is suitable for individual and small team projects

Although GitHub no longer provides a separate Automated Kanban template, the Kanban board still supports Agile workflows effectively through manual task management and issue tracking.

This makes it suitable for managing the AccessLearn project.
