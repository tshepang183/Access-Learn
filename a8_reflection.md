# Reflection – Assignment 8

## Challenges

One challenge was deciding how detailed the diagrams should be. Including too many states made diagrams complex, while too few reduced clarity.

Another challenge was aligning diagrams with previous Agile user stories and ensuring consistency with functional requirements.

## State vs Activity Diagrams

State diagrams focus on how an object changes over time (e.g., a lesson moving from not opened to completed).

Activity diagrams focus on workflows and processes (e.g., how a user downloads notes).

## Learning Outcome

This assignment improved understanding of system behavior and process modeling. It also showed how design diagrams support implementation and help visualize system functionality before development.
